Von Neumann's CA and close variations

JvN29*
John von Neumann's original 29-state CA.
Nobili32*
Renato Nobili's extension of von Neumann's 29-state CA to allow easier crossing of wires, leading to enormously simpler machines.
Hutton32*
This is a modification of Nobili32 by Tim Hutton, to allow simpler construction and rotational invariance.

Codd's CA and descendents

Codd*
Edgar F. Codd simplified von Neumann's CA to 8 states.
Codd-ICRA
A team of Hungarian researchers modified Codd's rules. Crossovers can be made with only nine cells, and gates can be constructed by passing 7-0 signals.
Devore*
John Devore altered Codd's ruleset to allow for more compact machines. Golly has a functioning self-replicator in this ruleset: Devore-rep.rle*. Adam Goucher made another replicator that uses a weak form of run-length encoding to compress the size of the tape: D-compressed-replicator.gz.
Codd2*
Tim Hutton realised that Codd's rule table required three extra transitions in order to be able to sheath large complex structures. With this rule, an implementation of Codd's self-replicating machine was created: Codd-self-rep.zip (1MB).

Self-replicating loops

Langtons-Loops*
Chris G. Langton extended Codd's rules to allow a totally novel form of simple self-replicator - the loop.
Byl-Loop*
J. Byl managed to reduce the size of Langton's Loop.
Chou-Reggia-1* Chou-Reggia-2*
A further reduction of Langton's Loops - down to just five cells. This is a modification of Byl's loop that doesn't even need an external sheath. However, the simplicity of it makes it difficult to identify it as a loop.
Tempesti*
Gianluca Tempesti's programmable loop that is capable of constructing various things inside itself.
Perrier*
Perrier added universal computation capabilities to Langton's loop by adding a program stack and an extensible data tape.
SDSR-Loop*
Hiroki Sayama introduced a change to Langton's Loops that caused dead loops to disappear, allowing live ones to reproduce further.
Evoloop*
Another loop from Sayama, that allows colliding loops to sometimes merge genetic content.
Goucher's Loops
A 24-state version of Langton's loops, that permits construction and computation universality, right turns as well as left turns, genome combination and competition.

Other computation rules

Banks-I* Banks-II* Banks-III* Banks-IV*
Edwin Roger Banks made CA that supported universal computation and construction, using fewer states than before. The first of these rules allows any finite computation to be calculated (equivalent to WireWorld). The second rule allows a finite starting configuration to have unbounded memory. The third rule is equivalent to the second rule, but it uses the Moore neighbourhood to reduce the number of states to just two. The fourth rule allows universal construction, hence replication.
WireWorld*
Brian Silverman's famous CA for electronic wiring.
WWE
A 17-state rule created by Alan Tennant to support universal computation and construction. Unlike the original WireWorld rule, this rule supports self-replication. Adam Goucher has included 3 of his replicators in the zip file.
MRM
A 4-state rule made by Paul Chapman to support Minsky Register Machines, up to and including universal MRMs. The supplied MRM calculates the nth prime.
WSwitch
A 5-state rule by Alan Tennant. The rule is based on WireWorld, but has been designed so that the construction of logic gates is impossible. Instead, pulse dividers are possible.

Lattice gases

HPP*
The HPP lattice gas can be simulated in the Margolus rule, but Brian Wylie showed how to simulate it with 16 states and the von Neumann neighbourhood. Tim Hutton added another 16 'reflection' states to allow the rule to function in a bounded, non-toroidal space.
DLA
Diffusion-limited aggregation using the HPP lattice gas. This model was studied to simulate the fractal growth characterised by electrolysis of copper sulphate.

Margolus neighbourhood

DLA-Margolus-emulated*
Diffusion-limited aggegation in the Margolus neighbourhood.
BBM-Margolus-emulated*
Ed Fredkin's Billiard Ball Model in the Margolus neighbourhood.
Sand-Margolus-emulated*
MCell's 'Sand' rule in the Margolus neighbourhood.

Langton's Ant and other Turing Machines

Langtons-Ant*
Langton's other famous system. An ant moves around an infinite universe, flipping the colour of the squares it lands on and turning left or right accordingly.
Iceskater
A rule by Jordan Goldstein. An explanation is included in the zip file.
Worm-1040512* Worm-1042015* Worm-1042020* Worm-1252121* Worm-1525115*
Dean Hickerson's implementation of Paterson's Worms. The hexagonal neighbourhood is emulated using the Moore neighbourhood.
BusyBeaver3 BusyBeaver5
Adam Goucher has made a program to convert m-state n-symbol Turing machines into rule tables. The two example machines are Busy Beavers, Turing machines that try to write as many '1's as possible before halting. To run each machine in Golly, seed the universe with a single cell of state 2.

Life-inspired CAs

LifeColor
A colored variant of the Game of Life, by Francois Boisson.
LifeOnTheEdge*
Franklin T. Adams-Watters described a CA in which all the action occurs on the edges of a square grid. Each edge can be on or off and has six neighbors, three at each end.
LifeOnTheSlope*
An implementation of the above rule, rotated 45 degrees so that only 2 live states are required.
HistoricalLife
An implementation of the classic game of life that highlights the cells that were ever alive. Rule by Dave Greene, translation to a table by Calcyman.
LifeHistory
A five-state extension of the HistoricalLife rule originally by Brice Due. The LifeHistory rule is included in Golly but the given zip file contains an optimized version of the HistoricalLife rule and several support scripts for both rules.
Life Pattern Emulators
Two rules, "Clouds" by Dean Hickerson and "Primes" by Adam Goucher, designed to simulate two of Dean's transcendental Life patterns. Included in the zip file are the rule tables, sample patterns and equivalent Life patterns.

Miscellaneous

Ed-rep*
In 1970, Terry Winograd proved that Fredkin's replicator CA (the parity rule B1357/S1357) could be extended to N states, as long as N is a prime number. Golly's demo pattern Ed-rep.rle* shows a 7-color photo of Ed Fredkin that replicates itself.
CyclicCA
A rule investigated by David Griffeath. This rule is a very basic CA that shows competition emerging from a random initial seed.
JustFriends
A non-totalistic 2D binary rule by David Bell.
TrickyBees
A chaotic rule by Alan Tennant where each state becomes progressively more inert in which patterns are surprisingly tricky to construct.
MazeSolver
A 13-state rule by Adam Goucher that solves any maze, using a flood fill method to locate the exit, before backtracking to the start to highlight the shortest possible route.
